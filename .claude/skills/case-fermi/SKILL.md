# フェルミ推定の方法論

面接官がフェルミお題を回す時に内部参照する手法集。**模範分解は候補者に開示しない。**

## 標準アプローチ（5ステップ）

1. **前提定義**: 対象範囲・地理・期間・定義の確認（例:「コンビニ売上 = 日本全国・年間・小売売上ベース」）
2. **アプローチ選択**: 需要側 / 供給側 / プロセスベース のどれで攻めるか宣言
3. **分解**: ロジックツリーで構造化（MECEに）
4. **数値投入**: 既知値 + 推定値で計算
5. **サニティチェック**: 別ルート / 桁 / 上限値で検証

候補者がこの流れを踏めていない場合、`case-coaching-techniques` の該当テンプレを使う。

## 分解の代表型

| 型 | いつ使う | 典型分解 |
|---|---|---|
| **人口ベース（需要側）** | 個人消費の市場規模 | 人口 × 利用率 × 頻度 × 単価 |
| **世帯ベース** | 耐久財・家電・住宅 | 世帯数 × 普及率 × 買替頻度 × 単価 |
| **面積ベース（供給側）** | 店舗・施設の集客 | 店舗面積 × 坪効率 × 営業日 |
| **施設数ベース（供給側）** | 業界全体の売上 | 店舗数 × 1店舗あたり売上 |
| **プロセスベース** | サービス・1日の利用者数 | 営業時間 ÷ 1人あたり所要時間 × 同時並行数 × 稼働率 |
| **ストック・フロー** | 保有台数・累積市場 | 新規流入 × 平均使用年数（フロー → ストック） |
| **時間ベース** | 1日/1年あたり何回起こるか | 対象人口 × 1人あたり頻度 × 期間 |

**面接官の役割**: 候補者がどの型を選んだか宣言させる。型を曖昧に始めたら止める。

## 需要側 vs 供給側の判断

- **需要側**: 顧客起点。市場の天井を知りたい時
- **供給側**: 提供能力起点。実際の販売量・収益を知りたい時
- **両側からの突合**: 高得点。供給側 ≦ 需要側 という制約も意識
- 面接官として「もう一方からも検証できますか？」を必ず1回は聞く

## 数値感覚の参考値（日本基準）

候補者が大きく桁外れの数字を使ったら指摘する。

- 日本人口: 約 **1.25億人**
- 世帯数: 約 **5,500万世帯**（平均世帯人員 約2.3人）
- 平均寿命: 男 **81歳** / 女 **87歳** → 概算 **84年**
- 年代分布ざっくり: 0-14歳 12%, 15-64歳 59%, 65歳以上 29%
- 就業者数: 約 **6,700万人**
- 東京都人口: 約 **1,400万人**
- 東京23区人口: 約 **970万人**
- 国土面積: 約 **38万 km²**（うち平野部約 25%）
- 1年: **365日 / 8,760時間**、稼働日 **約240日**
- 1日の起きている時間: 約 **16時間**
- コンビニ店舗数: 約 **56,000店**
- 小売業店舗数: 約 **100万店**
- GDP: 約 **600兆円**
- 1人あたり年間消費支出: 約 **130万円**
- 平均所得: 世帯 **約560万円** / 個人給与 **約460万円**

## サニティチェック作法

候補者の最終数値が出たら、面接官として必ず1つは投げる:

1. **桁チェック**: 「1兆円規模になりましたが、それは日本のGDP（600兆円）の何%にあたりますか？」
2. **上限チェック**: 「対象人口を全員カウントしても、その数になりますか？」
3. **別ルート検証**: 「供給側からも推定してみたら、近い値になりますか？」
4. **既知値との突合**: 「業界統計（公表値）と比べて妥当な範囲ですか？」（実数値を知っていれば内部で照合）
5. **頻度の妥当性**: 「1人が年に〇回というのは、自分の感覚と合っていますか？」

## つまずきポイント早見表（指摘の引き出し）

候補者がやりがちな失敗 → 投げる質問:

| 失敗 | 質問 |
|---|---|
| 切り口の根拠を言わずに分解開始 | 「なぜその切り口を選びましたか？」 |
| 利用率・頻度を「えいや」で置く | 「その利用率はどう推定しましたか？」 |
| 桁外れの数字に気づかない | 「その値、日本人口と比べてどうですか？」 |
| 単位を曖昧にする（円/月か円/年か） | 「単位を明示してください」 |
| 平均で全部丸める（属性差を無視） | 「全員同じ前提で大丈夫ですか？層に分けるべき？」 |
| 供給制約を無視（需要側だけ） | 「実際にそれだけ供給できますか？店舗・人手の制約は？」 |
| サニティチェックを自分からしない | 「妥当性、どう確認しますか？」 |
