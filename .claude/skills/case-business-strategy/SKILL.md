# ビジネスケース戦略（売上利益改善・市場参入）

面接官がビジネスケース／市場参入のお題を回す時に内部参照するフレーム集。**模範分解は候補者に開示しない。**

## フレーム選択の判断軸

| お題タイプ | 第一選択フレーム |
|---|---|
| 売上が落ちている / 売上を上げたい | 利益ツリー → 売上分解（客数×単価×頻度） + 3C |
| 利益が落ちている / コストを下げたい | 利益ツリー → コスト分解（固定費/変動費、バリューチェーン） |
| 新市場に参入すべきか | 市場魅力度 × 自社適合性 × 実行可能性 |
| 新規事業をやるか | 5C（市場/競合/自社/顧客/規制）+ 投資判断 |
| 競争環境を見たい | 5Forces |
| 製品/サービスをどう売るか | 4P + STP |

## 1. 利益ツリー（売上・利益改善の基本）

```
利益 = 売上 − コスト

売上 = 客数 × 客単価
     = (新規客数 + 既存客数) × 客単価
     = 既存客数 × リピート率 × 客単価
     = (店舗数 × 1店あたり客数) × 客単価
     = 市場規模 × 自社シェア

客単価 = 購買点数 × 1点あたり単価

コスト = 変動費 + 固定費
      = (原価率 × 売上) + 固定費
      = 仕入 + 物流 + 人件費 + 賃料 + 販促費 + ...
```

**候補者へのチェック**:
- 売上分解で「客数 × 単価」止まりなら、さらに分解できないか問う
- 「新規 vs 既存」「既存のリピート率」を意識しているか
- コスト側もMECEに分解できているか

## 2. 3C / 4P / STP

### 3C（市場理解）
- **Customer**: 誰が、何を、なぜ、いくらで、どこで買うか
- **Competitor**: 競合は誰か、強み弱み、シェア、最近の動き
- **Company**: 自社の強み弱み、リソース、過去の経緯

### 4P（マーケミックス）
- Product / Price / Place / Promotion

### STP
- Segmentation / Targeting / Positioning

## 3. 市場参入の判断フレーム（核）

### 市場魅力度 × 自社適合性 × 実行可能性

```
参入すべきか = 市場魅力度 × 自社適合性 × 実行可能性
                （これらが「掛け算」である点に注意）
```

1. **市場魅力度**
   - 市場規模（現在 / 将来）
   - 成長率
   - 収益性（利益率の水準）
   - 競争状況（5Forces）

2. **自社適合性**
   - 既存事業とのシナジー（顧客・チャネル・技術）
   - 強み（コア・ケイパビリティ）が活きるか
   - ブランドとの整合性

3. **実行可能性**
   - 参入障壁（規制 / 資本 / 技術 / ネットワーク効果）
   - 必要リソース（人・金・時間）
   - 撤退コスト

**面接官チェック**: 候補者がこの3軸の「どれか1つ」で判断していたら、他軸も問う。

### 参入モード

- 自前 / M&A / 提携・JV / ライセンス
- 判断軸: スピード × コントロール × 投資額 × リスク

## 4. 5Forces（競争環境分析）

1. 既存競合の脅威
2. 新規参入の脅威
3. 代替品の脅威
4. 供給者の交渉力
5. 買い手の交渉力

候補者がこれを言及せずに「競合がいない」と言ったら、代替品・新規参入の可能性を問う。

## 5. バリューチェーン（コスト改善・差別化検討）

```
主活動: 購買物流 → 製造 → 出荷物流 → マーケ販売 → サービス
支援活動: 全般管理 / 人事 / 技術開発 / 調達
```

各機能で「コスト削減 or 差別化」の余地を見る。

## 6. 打ち手の評価軸

候補者が打ち手を出してきたら以下で評価させる:

| 軸 | 質問 |
|---|---|
| インパクト | 売上/利益への定量効果は？ |
| 実行容易性 | 必要リソース・期間は？ |
| リスク | 失敗時の損失は？撤退できるか？ |
| 競合反応 | 競合が同じ手を打ったらどうなる？ |
| 持続性 | 短期効果か長期効果か？模倣されないか？ |

優先順位は **インパクト × 実行容易性** の2軸でマトリクス化させると良い。

## 7. So what / Now what（結論の出し方）

候補者の最終回答で必ず確認:
- **結論ファースト**で言えているか
- 結論 + 根拠3つ + リスク + 次のアクション の構造
- 「やる/やらない」を曖昧にしていないか
- 数値（インパクト試算）が結論に紐づいているか

## 8. 候補者がやりがちな失敗 → 投げる質問

| 失敗 | 質問 |
|---|---|
| いきなり打ち手から議論 | 「課題の特定はできていますか？まず現状を把握しませんか？」 |
| 売上分解が「単価×客数」で止まる | 「もう一段分解できますか？新規/既存、頻度は？」 |
| 競合・代替を見ない | 「競合の動き、代替手段はどうですか？」 |
| 自社の強みを忘れる | 「これは自社だからやる意味がありますか？他社でもできますか？」 |
| 定量インパクトなし | 「具体的に売上いくら、利益いくら見込みますか？」 |
| リスクを言わない | 「想定リスクは？失敗時の損失と撤退の判断軸は？」 |
| 市場参入判断で「魅力度だけ」で結論 | 「自社適合性と実行可能性も評価しましたか？」 |
| 全枝を均等に掘ろうとする | 「優先順位はインパクト × 実行容易性で見るとどこから？」 |
